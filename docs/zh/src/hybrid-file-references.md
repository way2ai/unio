# 混合文件引用

混合 CLI 支持在提示中使用 `@path` 进行轻量级文件引用。当 `@` 出现在输入行首或紧跟空白字符后时触发建议，因此 `@README.md` 和 `inspect @src/main.rs` 均有效。

文件建议来自后台线程维护的内存工作区索引。索引在遍历目录树时避免阻塞按键，并过滤噪声路径：

- 遵循 `.gitignore` 和 `.npmignore` 的简单规则；
- 跳过隐藏目录（如 `.git`）；
- 跳过大型生成目录（如 `node_modules`、`target`、`dist`、`build`、`.next`、`.cache`、`coverage`）；
- 仅向输入建议列表暴露相对工作区文件路径。

建议排序优先匹配用户最可能的目标：精确基名匹配优先，其次是精确相对路径匹配、基名前缀匹配、路径前缀匹配、较短路径，最后按字典序排列。

交互过滤必须保持在内存中，避免大量内存分配。每次按键时 UI 不得递归扫描文件系统或对全部匹配集排序。索引预先规范化小写路径和基名，建议查询仅保留最优的 50 个候选。后台刷新目前采用慢轮询，而非逐帧扫描。

索引是轻量级的交互提示来源，不是安全边界，也不能替代工具级别的路径验证。
