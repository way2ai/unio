# How Long Contexts Fail 总结

原文链接：https://www.dbreunig.com/2025/06/22/how-contexts-fail-and-how-to-fix-them.html#context-distraction

这篇文章的核心观点是：长上下文窗口不是“越大越好”。把所有文档、工具、历史记录、指令都塞进 prompt，反而会让 agent 在几种方式上失灵。

## 主要内容

1. Context Poisoning（上下文中毒）

   一旦幻觉或错误信息进入上下文，模型会不断引用它，后续推理被污染。例子是 Gemini 玩 Pokemon 时，如果目标状态被错误信息污染，就会反复追求不可能完成的目标。

2. Context Distraction（上下文分心）

   这是链接锚点指向的部分。上下文太长时，模型会过度依赖历史内容，而不是调用训练中学到的能力来重新规划。文章提到 Gemini 在上下文超过约 100k tokens 后，更倾向于重复历史动作，而不是生成新策略。Databricks 的研究也显示，一些模型在远未填满上下文窗口时正确率就开始下降。

3. Context Confusion（上下文混淆）

   无关内容也会影响模型。比如把太多工具定义塞进上下文，模型可能被不相关工具干扰，甚至调用不该调用的工具。作者强调：只要放进上下文，模型就不得不“看见”并考虑它。

4. Context Clash（上下文冲突）

   上下文中不同来源的信息互相矛盾，会严重破坏推理。多轮对话尤其容易出现这个问题，因为早期不完整或错误的回答会留在上下文里，影响最终答案。

## 作者的结论

百万 token 上下文确实很强，但它主要适合检索和总结，不等于可以无脑塞入所有信息。Agent 最容易受影响，因为它们天然会积累工具调用、文档、历史消息和中间推理。

真正关键的能力不是拥有超长上下文，而是管理上下文：控制放入什么、何时放入、如何隔离错误或无关信息。
